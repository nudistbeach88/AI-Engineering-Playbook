# Changelog

All notable changes to the AI Engineering Playbook are recorded here. Versions describe methodology releases, not application releases, and use semantic versioning.

## [Unreleased]

### Added

- Added a domain-neutral Core Intelligence Architecture covering acquisition, canonical entities, evidence, relationships, explainable conclusions, confidence, human decisions, learning events, intelligence completeness, recommendations, and delivery views.
- Added a six-level capability maturity model from data storage through bounded autonomous recommendations.

### Changed

- Linked the new architecture from the parent build methodology and repository index.
- Clarified that specialized architecture patterns remain optional and that simulated intelligence must not be presented as production capability.
- Updated the roadmap to reflect the approved architecture while continuing to defer organization-wide maturity scoring.

## [0.2.1] - 2026-07-11

### Changed

- Expanded the Teaching Methodology into the Playbook-wide Communication & Teaching Methodology.
- Established Mental Model First as the default philosophy for engineering explanations, collaboration, documentation, and reviews.
- Added a concise review sequence designed to build understanding of engineering decisions before presenting implementation detail.

## [0.2.0] - 2026-07-11

### Changed

- Adopted AI Engineering Playbook as the canonical methodology name while preserving Nathan's ownership and `.ai/BUILD_PLAYBOOK.md` as the core build methodology.
- Replaced the single source hierarchy with a concise distinction between descriptive truth and normative authority.
- Made constitutional formality and technical teaching increments proportional to context.
- Clarified how project lessons move through review, implementation, versioning, and later evaluation.
- Added a risk-proportional quality gate for security, privacy, authentication and authorization, permissions, credentials, destructive operations, stored data, migration, integrity, and user exposure.
- Added lightweight evidence traceability guidance for future anti-patterns.
- Separated lesson structure from engineering history, added optional evidence references, and defined lesson statuses.
- Aligned the roadmap and navigation with the Version 0.2.0 methodology.

### Corrected

- Removed the Version 0.1.0 illustrative lessons from the active ledger because they were examples, not accepted project evidence.
- Corrected the Version 0.1.0 history below to describe only files tracked by Git; empty `templates/` and `examples/` directories were not part of the committed tree.

## [0.1.0] - 2026-07-11

### Added

- Initial committed repository structure containing seven Markdown documents across `.ai/`, `docs/`, and the repository root.
- Parent build playbook with the foundational engineering framework.
- Teaching Methodology with the Mental Model First sequence and Bill Nye Protocol.
- Quality gates for implementation and completion evidence.
- Initial catalog of recurring AI engineering anti-patterns.
- Lessons ledger with illustrative format examples, a concise repository introduction, an evidence-led roadmap, and release history.

### Intentionally Deferred

- Tracked templates, example artifacts, specialized playbooks, and automation pending evidence from real projects.
